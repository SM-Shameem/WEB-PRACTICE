SLNO GLOBAL-ATTRIBUTES VALUE BROWSER USAGE/EXCEPTION
01 accesskey letter/digit
(singel)		 all # specifies a shortcut key to activate/focus an element
# the access attribute value must be a single character (letter/digit)
# way of accessign shortcut key element is varying in different browsers and system
# alt+accesskey (windows/linux-chrome), control+alt+accesskey (mac-chrome)
02 class className
(one/more)		 all # specifies one or more classnames for an elements
# class attribute used to point to (class) in css/used by JavaScript via DOM
# specifies one or more class names for an element
# multiple class names are seperated with space, to combine several css for element
# naming rules - must begin with a leter, then (leter/digits/hypens/underscore)
03 contenteditable true/false version # specifies whether the content of an element is editable or not
# if attribute is not set on element then inherit from its parent
04 data-* somevalue
(as a string)		 version # used to store custom data private to the page or application.
# the (data-*) attributes gives us the ability to embed custom data on elements
# the stored data can then be used in the page's JavaScript
(without any Ajax calls or Server-side database queries)
# custom attributes prefixed with "data-" will be completely ignored by user agent
# att-name must lowercase and at-least one char long string with prefix "data-"
05 dir ltr/rtl/auto all # specifies the text direction for the content in an element
# in-case of (auto) browser figure out the direction, based on content (unknow)
06 draggable true/false/auto version # specifies whether an element is draggable or not
# the draggable attribute is often used in drag and drop operation
# auto - is browser default / links and images are draggable by defautl
07 hidden only attribute version # specifies that an element is not yet, or is no longer, relevant.
# browser should not display elements that have the hidden attribute
# the (hidden) attribute is a boolean attribute
# it can also be used to keep a user from seeing an element until some other
condition has been met. then javascript could remove (hidden) attribute
08 id idName all # specifies a unique id for an element (value must be unique within the document)
# is most used to point to style in css/used by JavaScript via DOM
# naming rules - must contain at least one character (space not allowed)
09 lang language_code all # specifies the language of the element's content
# common examples are (en) for English, (es) for Spanish, and so on
10 spellcheck true/false version # whether element is to have its spelling and grammar checked or not
# can be checked (text-value in (input)/in (textarea)/text in editable element)
11 style style_definitions all # specifies an inline css style for an element.
# it override any style set globally (internal or external).
# can be used on any element but it is not necessarily useful
# one or more (css) properties and values separated by semicolones
12 tabindex number all # specifies the tabbing order of an element.
# can be used on any element but it is not necessarily useful
# specifies the tabbing order of the element (1 is first)
13 title text all # specifies extra information about an element.
# can be used on any element but it is nto necessarily useful
# a tooltip text for an element (when mouse over the element)
14 translate yes/no no # whether the content of an element should be translated or not s
