48 method # get
# post		 form all # specifies the http method to use when sending form-data

# form-data can be sent as url (method="get")
# form-data can be sent as HTTP post transction (method="post")
# notes on GET -
> appends form-data into the url in (name/value) pairs
> the length of a url is limited (about 3000 characters)
> never use GET to send sensitive data (will be visible in the URL)
> useful for form submissions where user want to bookmark the result
> GET is better for non-secure data, like query strings in Google
# notes on POST -
> appends form-data inside (body) of HTTP request
data not shown in URL
> has no size limitations
> form submissions with POST cannot be bookmarked
