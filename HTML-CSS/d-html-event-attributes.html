SLNO WINDOW-EVENTS BELONGS-TO BROWSER USAGE/EXCEPTION
01 onafterprint body version # script runs after document is printed or print dialogue box has been closed.
# IE-Edge run berofe print Dialog box, instead of after.
# used together onbeforepr
02 onbeforeprint body version # script runs before document is printed or before print dialogue box appears
# used together onbeforepr
03 onbeforeunload body opera version # script runs when document is about to be unloaded
# display a message in a dialog box to stay or leave the current page
# the default message cannot remove but is differe in browsers
# you can write a custom message but (Firefox) only show the default message
04 onerror body all # script runs when an error occurs, while loading an external file
# support, (img/object/script/link) elements also (audio/embed) elements
# support, (input type) value (image)
05 onhashchange body version # script runs when there has been changes to the anchor part of the url,
begins with a (#) symbol
# what an anchor part is: the current URL is (www.example.com/test.htm#part2)
# The anchor part of this URL would be (#part2)
# To invoke this event, you can -
1. Change anchor part by (location.hash/location.href) property of Location Object
2. Navigate to current page with a different bookmark (Use "back" or "forward")
3. Click on a link to a bookmark anchor
06 onload body all # script runs when page/object is finished loading (image, script, css, etc)
# is most often used within (body) element
# however, it can be used on other elements as well
# used to check visitor's browser and version, and web page version based on info
# support, (body/frame/frameset/iframe/img/link/script/style)
# support, (input type) value (image)
07 onmessage body version # script runs when the message is triggered
08 onoffline body except Firefox # script runs when the browser starts to work offline
# onoffline is opposite of ononline
09 ononline body except Firefox # script runs when the browser starts to work online
# ononline is opposite of onoffline
10 onpagehide body version # script runs when a user navigates away from a page
11 onpageshow body version # script runs when a user navigates to a page
# onpageshow is similar to onload, except, it occurs after onload when first loads
# it occurs every time of page loading, whereas onload not occur (load from cache)
12 onpopstate body version # script runs when the window's history changes
13 onresize body all # script runs when the browser window is being resized
14 onstorage body version # script runs when a web storage area is updated
15 onunload body all # script runs when a page has unloaded or browser closed
# occurs when user navigates away from page (by clicking link, submit form, etc)
# on reload a page, onunload is triggered also with onload
# due to different browser settings, event may not always work as expected
SLNO FORM-EVENTS BELONGS-TO BROWSER USAGE/EXCEPTION
  events triggered by actions inside a HTML-Form (applies to almost all HTML elmenets also)
01 onblur visible elements all # script runs when the element loses focus
# Onblur is most often used with form validation (user leaves a form field)
# onblur attribute is the opposite of the (onfocus)
# support, all HTML elements,
except (base/bdo/br/head/html/iframe/meta/param/script/style/title)
02 onchange visible elements all

# script runs when the value to the element is changed
# onchange is similar to (oninput), difference, it occurs immediately after the value
changed, while (onchanges) occurs after loseing focus
# support, (input type) value (checkbox/file/password/radio/range/search/text)
# support, (select and textarea) element
03 oncontextmenu visible elements all

# script runs when a context menu is triggered

04 onfocus visible elements all # script runs when the element gets focus
# onfocus is most often used with (input/select/a)
# onfocus is oposite of (onblur)
# support all HTML-elements,
except (base/bdo/br/head/html/iframe/meta/param/script/style/title)
05 oninput visible elements version # script runs when the element gets user input
# script runs when the value of an (input/textarea) is changed
# oninput is similar to (onchange), difference, it occurs immediately after the value
changed, while (onchanges) occurs after loseing focus
# support, (input type) value (password/search/text)
# support, (textarea) also
06 oninvalid visible elements version # script runs when the element is invalid (input element)
# the input field is invalid if the required attribute is set and the field is empty
# support, (input) element
07 onreset form all # script runs when a reset button in a form is clicked
08 onsearch input version # script runs when the user writes something in a search field
# support, (input type) value (search)
09 onselect visible elements all # script runs when the element gets selected
# support, (input type) value (file, password, text)
# support, (textarea) element also
10 onsubmit form all # script runs when a form is submitted
SLNO KEYBOARD-EVENTS BELONGS-TO BROWSER DEFINES/SPECIFIES
01 onkeydown visible elements all # script runs when a user is pressing a key (on the keyboard)
# the order of events related to the (onkeydown) event - onkeypress/onkeyup
# support, all elements
# except, (base/bdo/br/head/html/iframe/meta/param/script/style/title)
02 onkeypress visible elements all # script runs when a user presses a key (on the keyboard)
# the order of events related to the (onkeypress) event - onkeydown/onkeyup
# (onkeypress) is not fired for all keys (e.g. alt, ctrl, shift, esc) in all browsers
# to detect only whether user has pressed key, use (onkeydown), it works for all keys
# support, all elements
# except, (base/bdo/br/head/html/iframe/meta/param/script/style/title)
03 onkeyup visible elements all # script runs when a user releases a key (on the keyboard)
# the order of events related to the (onkeyup) event - onkeydown/onkeypress
# support, all elements
# except, (base/bdo/br/head/html/iframe/meta/param/script/style/title)

SLNO DRAG-EVENTS BELONGS-TO BROWSER DEFINES/SPECIFIES
01 ondrag visible elements version # script runs when the element or text selection is being dragged
# while dragging an element, the ondrag event fires evry 350 milliseconds
# drag and drop is common feature in HTML5 -
> when garb an object and drags it different location
> (link/image) default, to make an element draggable, use global (draggable)
> many (events) are used/can occure, in different stages of (drag/drop) operation
1. events fired on the draggable target (the source element) - ondragstart/ondragend
2. events fired on the drop target - ondragenter/ondragover/ondragleave/ondrop
02 ondragend visible elements version # script runs at the end of a drag operation (element/text)
# drag and drop is common feature in HTML5 - rest of, as same as (ondrag)
03 ondragenter visible elements version # script runs when an element has been dragged to a vaild drop target (element/text)
# (ondragenter/ondragleave) help to understand, draggable element (leave drop-target)
# drag and drop is common feature in HTML5 - rest of, as same as (ondrag)
04 ondragleave visible elements version # script runs when an element leaves a valid drop target (element/text-section)
# (ondragenter/ondragleave) help to understand, draggable element (enter drop-target)
# drag and drop is common feature in HTML5 - rest of, as same as (ondrag)
05 ondragover visible elements version # script runs when an element is being dragged over a valid drop target (element/text)
# by default, data/elements cannot be dropped in other elements. to allow a drop, must
prevent the default handling of the element boy calling the (event.preventDefault())
method for the (ondragover) attribute
# drag and drop is common feature in HTML5 - rest of, as same as (ondrag)
06 ondragstart visible elements version # script runs at the start of a drag operation (element/text)
# drag and drop is common feature in HTML5 - rest of, as same as (ondrag)
07 ondrop visible elements version # script runs when dragged element/text is being dropped on a valid drop target
# drag and drop is common feature in HTML5 - rest of, as same as (ondrag)
08 onscroll visible elements all # script runs when an element's scrollbar is being scrolled
# use the (css - overflow) style property to create a scrollbar for an element
# supported, (address/blockquote/body/caption/center/dd/dir/div/dl/dt/fieldset/form)
# supported, (h1-h6/html/li/menu/object/ol/p/pre/select/tbody/textarea/tfoot/thead/ul)
