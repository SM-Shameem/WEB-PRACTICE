# javascript strings 
- a javascript strings stores a series of characters like ("john doe")
- a string can be any text inside (double or single) quotes 
  like > var carName1 = "volvo xc60";
  like > var carName2 = 'volvo xc60';
- string indexes are zero-based: the first character is in position 0, the second in 1, and so on 

# String 
- normally, javascript strings are primitive values, created form literals, but strings can also be defined as 
  objects with the keyword (new) 
  like > var firstName = new String("john");
- don't create strings as objects. it slows down executing speed. the (new) keyword complicates the code.
  this can produce some unexpected results 
   
# padStart and padEnd
- ECMAScript 2017 added two String methods (padStart and padEnd) to support padding at the beginning and 
  at the end of a string 
  like > let str = "5"; str = str.padStart(4, 0); 
  like > let str = "5"; str = str.padEnd(4, 0);
- string padding is not supported IE but firefox and safari were the first browser with support for padding

# property access - ECMAScript 5 (2009) allows property access [] on strings 
  like > var str = "hello world"; str[0]; 
- property access might be a little (unpredictable) 
  - it does not work in IE7 or earlier 
  - it makes strings look like arrays (but they are not) 
  - if no characer is found. [] returns undefined. while charAt() returns an empty string 
  - it is ready only str[0] = "A" gives on error (but does not work)
  like > var str = "hello world"; str[0] = "a"; // gives no error, but does not work 
- if you want to work with a string as an array, you can convert it to an array

# string properties and methods with HTML wrapper methods 
  example > var str = "john doe";
- primitive values, like ("john doe"), cannot have properties or methods (because they are not objects) 
- but with javascript, methods and properties are also available to primitive values, because javascript treats 
  primitive values as objects when executing methods and properties 
- all string methods return a new value. they do not change the original variable 

- the HTML wrapper methods return the string wrapped inside the appropriate HTML tag 
- these are not standard methods, and may not work as expected in all browsers

# charAt() - strVar.charAt(index) - ECMAScript1 (extracting string characters)
  parameter > (index) - required, an integer representing the index of the character you want to return 
  return    > a string, representing the character at the specified index, or an empty string if the index 
              number is not found 
  example   > var str = "string one"; var res = str.charAt(0);
  example   > var str = "string two"; var res = str.charAt(str.length-1);
- The charAt() method returns the character at the specified index in a string.
- The index of the first character is 0, the second character is 1, and so on.
- Tip: The index of the last character in a string is string.length-1, the second last character is 
  string.length-2, and so on (See "More Examples").
  
# charCodeAt() - strVar.charCodeAt(index) - ECMAScript1 (extracting string characters)
  parameter > (index) - optional, a number representing the index of the character you want to return 
              if not specified, default value is 0
  return    > a number, representing the unicode of the character at the specified index 
              note, this method returns (NaN) if there is no character at the specified index, 
              or if the index is less than 0
  example   > var str = "string one"; var n = str.charcodeAt(0); 
  example   > var str = "string two"; var n = str.charCodeAt(str.length - 1);
- The charCodeAt() method returns the Unicode of the character at the specified index in a string.
- The index of the first character is 0, the second character 1, and so on.
- Tip: You can use the charCodeAt() method together with the length property to return the Unicode of the last 
  character in a string. The index of the last character is -1, the second last character is -2, 
  and so on (See Example below).
- Tip: For more information about Unicode Character Sets, visit our HTML Character Sets reference.

# slice() - strVar.slice(start, end) - ECMAScript1 (one of 3 methods for extracting string parts)
  parameter > (start) - required. the position where to begin the extraction. first character is at position 0
  parameter > (end) - optional. the position (up to, but not including) where to end the extraction. 
                      if omitted, slice() selects all charactes from the start-position to the end of the string 
                    - if you omit the second parameter, the method will slice out the rest of the string 
                    - or counting from the end
  return    > a string, representing the extracted part of the string 
  example   > var str = "Hello world!"; var res = str.slice(0, 5);
  example   > var str = "Hello world!"; var res = str.slice(0);
  example   > var str = "Hello world!"; var res = str.slice(3);
  example   > var str = "Hello world!"; var res = str.slice(3, 8);
  example   > var str = "Hello world!"; var res = str.slice(0, 1);
  example   > var str = "Hello world!"; var res = str.slice(-1);
- The slice() method extracts parts of a string and returns the extracted parts in a new string.
- Use the start and end parameters to specify the part of the string you want to extract.
- The first character has the position 0, the second has position 1, and so on.
- Tip: Use a negative number to select from the end of the string.

# substr() - strVar.substr(start, length) - ECMAScript1 (one of 3 methods for extracting string parts)
  parameter > (start) - required. the position where to start the extraction. first character is at index 0.
              start is positive and greater than, or equal, to the length of the string, returns an empty string
              if start is negative, substr() uses it as a character index from the end of the string 
              if start is negative or larger than the length of the string, start is set to 0 
  parameter > (length) - optional. number of characters to extract. if omitted, it extracts the rest of string 
  return    > a new string, containing the extracted part of the text. if length is 0 or negative, an empty 
              string is returned 
  example   > var str = "Hello world!"; var res = str.substr(1, 4); 
  example   > var str = "Hello world!"; var res = str.substr(2); 
  example   > var str = "Hello world!"; var res = str.substr(0, 1); 
  example   > var str = "Hello world!"; var res = str.substr(11, 1); 
- The substr() method extracts parts of a string, beginning at the character at the specified position, and
  returns the specified number of characters.
- Tip: To extract characters from the end of the string, use a negative start number (This does not work in IE 8)
- Note: The substr() method does not change the original string.

# substring() - strVar.substring(start, end); - ECMAScript1 (one of 3 methods for extracting string parts)
  parameter > (start) - required. the position where to start the extraction. first character is at index 0 
  parameter > (end) - optional. the position (up to, but not including) where to end the extraction. 
              if omitted, it extracts the rest of the string 
  return    > a new string containing the extracted characters 
  example   > var str = "Hello world!"; var res = str.substring(1, 4);
  example   > var str = "Hello world!"; var res = str.substring(2); 
  example   > var str = "Hello world!"; var res = str.substring(4, 1);
  example   > var str = "Hello world!"; var res = str.substring(-3);
  example   > var str = "Hello world!"; var res = str.substring(0, 1); 
  example > var str = "Hello world!"; var res = str.substring(str.length - 1, str.length);
- substring() method extracts characters from string, between two specified indices, and returns new sub string.
- This method extracts the characters in a string between "start" and "end", not including "end" itself.
- If "start" is greater than "end", this method will swap the two arguments, 
  meaning str.substring(1, 4) == str.substring(4, 1).
- If either "start" or "end" is less than 0, it is treated as if it were 0.
- Note: The substring() method does not change the original string. 

# indexOf() - strVar.indexOf(searchValue, start) - ECMAScript1 (finding a string in a string)
  parameter > (searchValue) - required. the string to search for 
  parameter > (start) - optional, default 0. at which position to start the search
  return    > a number, representing th eposition where the specified search-value occurs for the first time,
              or (-1) if it never occurs
  example   > var str = "Hello world, welcome to the universe."; var n = str.indexOf("welcome");
  example   > var str = "Hello world, welcome to the universe."; var n = str.indexOf("e");
  example   > var str = "Hello world, welcome to the universe."; var n = str.indexOf("e", 5);
- The indexOf() method returns the position of the first occurrence of a specified value in a string.
- This method returns (-1) if the value to search for never occurs.
- Note: The indexOf() method is case sensitive.
- Tip: Also look at the lastIndexOf() method.
- notice, the two methods (indexOf) and (serch), are equal. they accept the same arguments (parameters), 
  and returns the same value, but the two methods are NOT equal, these are the difference - 
  - the (search) method cannot take a second start position argument 
  - the (indexOf) method cannot take powerful search values (regular expressions)

# lsatIndexOf() - strVar.lastIndexOf(searchValue, start) - ECMAScript1 (finding a string in a string)
  parameter > (searchValue) - required. the string to search for 
  parameter > (start) - optional. the position where to start the search (searching backwards). if omitted, the 
              default value is the length of the string 
  return    > a number, representing the position where the specified search-value occurs for the last time, or 
              (-1) if it never occurs               
  example   > var str = "Hello planet earth, you are a great planet."; var n = str.lastIndexOf("planet");
  example   > var str = "Hello planet earth, you are a great planet."; var n = str.lastIndexOf("planet", 20);
- The lastIndexOf() method returns the position of the last occurrence of a specified value in a string.
- Note: The string is searched from the end to the beginning, but returns the index starting at at position 0.
- This method returns -1 if the value to search for never occurs.
- Note: The lastIndexOf() method is case sensitive!
- Tip: Also look at the indexOf() method.

# search() - strVar.search(searchValue) - ECMAScript1 (searcing for a string in a string)
  parameter > (searchValue) - required. a regular expression. a string will automatically be converted to a regExp
  return    > a number, representing the position of the first occurrence of the specified searchValue, or 
              (-1) if no match is found 
  example   > var str = "Visit W3Schools!"; var n = str.search("W3Schools");
  example   > var str = "Mr. Blue has a blue house"; var n = str.search("blue");
  example   > var str = "Mr. Blue has a blue house"; var n = str.search(/blue/i);
- The search() method searches a string for a specified value, and returns the position of the match.
- The search value can be string or a regular expression.
- This method returns -1 if no match is found.
- Read more about regular expressions in our RegExp Tutorial and our RegExp Object Reference.
- notice, the two methods (indexOf) and (serch), are equal. they accept the same arguments (parameters), 
  and returns the same value, but the two methods are NOT equal, these are the difference - 
  - the (search) method cannot take a second start position argument 
  - the (indexOf) method cannot take powerful search values (regular expressions)

# replace() - strVar.replace(searchValue, newValue) - ECMAScript1 (replacing string content)
  parameter > (searchValue) - required. the value, or regExp, that will be replaced by the new value 
  parameter > (newValue) - required. the value to replace the search value with 
  return    > a new string. where the specified value(s) has been replaced by the new value 
  example   > var str = "Visit Microsoft!"; var res = str.replace("Microsoft", "W3Schools");
  example   > var str = "Mr Blue has a blue house and a blue car"; var res = str.replace(/blue/g, "red");
  example   > var str = "Mr Blue has a blue house and a blue car"; var res = str.replace(/blue/gi, "red");
  example   > var str = "Mr Blue has a blue house and a blue car"; 
              var res = str.replace(/blue|house|car/gi, function (x) { return x.toUpperCase(); }); 
- The replace() method searches a string for a specified value, or a regular expression, and returns a new string 
  where the specified values are replaced.
- Note: If you are replacing a value (and not a RegExp), only first instance of the value will be replaced. 
  To replace all occurrences of a specified value, use the global (g) modifier (see "More Examples" below).
- Read more about regular expressions in our RegExp Tutorial and our RegExp Object Reference.
- This method does not change the original string.

- The split() method is used to split a string into an array of substrings, and returns the new array.
- Tip: If an empty string ("") is used as the separator, the string is split between each character.
- Note: The split() method does not change the original string.
