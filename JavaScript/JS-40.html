introduction
--------------------------------------------------------------------------------
# a regular expression is an object that describes a pattern of characters 
# regular expressions are used to perform pattern-matching and (search-and-replace) functions on (text) 
  syntax > /pattern/modifiers; 
  like   > var patt = /w3schools/i
- example explained 
  > /w3schools/1 - is a regular expression 
  > w3schools    - is a pattern (to be used in a search) 
  > i            - is a modifier (modifiers the search to be case-insensitive) 
--------------------------------------------------------------------------------
# there are three (03) modifiers
# modifieres are used to perform case-insensitive and global searches 
01. g - Perform a global match (find all matches rather than stopping after the first match)
02. i - Perform case-insensitive matching
03. m - Perform multiline matching
--------------------------------------------------------------------------------
# there are five (05) brackets
# Brackets are used to find a range of characters
01. [abc]  - Find any character between the brackets
02. [^abc] - Find any character NOT between the brackets
03. [0-9]  - Find any character between the brackets (any digit)
04. [^0-9] - Find any character NOT between the brackets (any non-digit)
05. (x|y)  - Find any of the alternatives specified
--------------------------------------------------------------------------------
# there are eighteen (18) metacharacters
# Metacharacters are characters with a special meaning:
01. .      - Find a single character, except newline or line terminator
02. \w     - Find a word character
03. \W     - Find a non-word character
04. \d     - Find a digit
05. \D     - Find a non-digit character
06. \s     - Find a whitespace character
07. \S     - Find a non-whitespace character
08. \b     - Find a match at the beginning/end of a word, beginning like this: \bHI, end like this: HI\b
09. \B     - Find a match, but not at the beginning/end of a word
10. \0     - Find a NULL character
11. \n     - Find a new line character
12. \f     - Find a form feed character
13. \r     - Find a carriage return character
14. \t     - Find a tab character
15. \v     - Find a vertical tab character
16. \xxx   - Find the character specified by an octal number xxx
17. \xdd   - Find the character specified by a hexadecimal number dd
18. \udddd - Find the Unicode character specified by a hexadecimal number dddd
--------------------------------------------------------------------------------
# there are ten (10) Quantifiers
01. n+     - Matches any string that contains at least one n
02. n*     - Matches any string that contains zero or more occurrences of n
03. n?     - Matches any string that contains zero or one occurrences of n
04. n{X}   - Matches any string that contains a sequence of X n's
05. n{X,Y} - Matches any string that contains a sequence of X to Y n's
06. n{X,}  - Matches any string that contains a sequence of at least X n's
07. n$     - Matches any string with n at the end of it
08. ^n     - Matches any string with n at the beginning of it
09. ?=n    - Matches any string that is followed by a specific string n
10. ?!n    - Matches any string that is not followed by a specific string n
--------------------------------------------------------------------------------
# there are five (05) RegExp Object Properties
# source 	Returns the text of the RegExp pattern
01. constructor - Returns the function that created the RegExp object's prototype
02. global      - Checks whether the "g" modifier is set
03. ignoreCase  - Checks whether the "i" modifier is set
04. lastIndex   - Specifies the index at which to start the next match
05. multiline   - Checks whether the "m" modifier is set
--------------------------------------------------------------------------------
# there are four (04) RegExp Object Methods
01. compile()  - Deprecated in version 1.5. Compiles a regular expression
02. exec()     - Tests for a match in a string. Returns the first match
03. test()     - Tests for a match in a string. Returns true or false
04. toString() - Returns the string value of the regular expression

--------------------------------------------------------------------------------
modifiers
--------------------------------------------------------------------------------
# (g) - new RegExp("regexp", "g") or simply /regexp/g - support all
  example > var str = "Is this all there is?"; var patt1 = /is/gi; 
  // do a global, case-insensitive search for is
- The g modifier is used to perform a global match (find all matches rather than stopping after the first match).
- Tip: To perform a global, case-insensitive search, use this modifier together with the "i" modifier.
- Tip: Use the global property to specify whether or not the g modifier is set.

# (i) - new RegExp("regexp", "i") or /regexp/i - support all 
  example > var str = "Visit W3Schools"; var patt1 = /w3schools/i;
  // do a case-insensitive search for "w3schools" in a string
- The i modifier is used to perform case-insensitive matching.
- Tip: Use the ignoreCase property to check whether or not the "i" modifier is set.

# (m) - new RegExp("regexp", "m") or simply /regexp/m - support all 
  example > var str = "\nIs th\nis it?"; var patt1 = /^is/m;
  // Do a multiline search for "is" at the beginning of each line in a string:
- The m modifier is used to perform a multi-line match.
- The m modifier treat beginning (^) and end ($) characters to match the beginning or end of each line of a 
  string (delimited by \n or \r), rather than just the beginning or end of the string.
- Note: The m modifier is case-sensitive and will stop the search after the first match. To perform a global, 
  case-insensitive, multi-line search, use this modifier together with "g" and "i".
- Tip: Use the multi-line property to specify whether or not the m modifier is set.

# [abc] - new RegExp("[abc]") or simply /[abc]/ - support all
  syntax with modifiers > new RegExp("[abc]", "g") or simply /[abc]/g
  example > Do a global search for the characters "i" and "s" in a string
  var str = "Do you know if this is all there is?"; var patt1 = /[is]/gi; 
  example > Do a global search for the character-span from lowercase "a" to lowercase "h" in a string:
  var str = "Is this all there is?";
  var patt1 = /[a-h]/g;
  example > Do a global search for the character-span from uppercase "A" to uppercase "E":
  var str = "I SCREAM FOR ICE CREAM!";
  var patt1 = /[A-E]/g; 
  example > Do a global search for the character-span from uppercase "A" to lowercase "e" 
            (will search for all uppercase letters, but only lowercase letters from a to e.)
  var str = "I Scream For Ice Cream, is that OK?!";
  var patt1 = /[A-e]/g; 
  example > Do a global, case-insensitive search for the character-span [a-s]:
  var str = "I Scream For Ice Cream, is that OK?!";
  var patt1 = /[a-s]/gi; 
  example > A demonstration of "g" and "gi"-search for characters:
  var str = "THIS This this";
  var patt1 = /[THIS]/g;
  var str = "THIS This this";
  var patt1 = /[THIS]/gi; 
- The [abc] expression is used to find any character between the brackets.
- The characters inside the brackets can be any characters or span of characters:
  > [abcde..] - Any character between the brackets
  > [A-Z] - Any character from uppercase A to uppercase Z
  > [a-z] - Any character from lowercase a to lowercase z
  > [A-z ]- Any character from uppercase A to lowercase z
- Tip: Use the [^abc] expression to find any character NOT between the brackets.

# [^abc] - new RegExp("[^xyz]") or simply /[^xyz]/ - support all 
  syntax with modifiers > new RegExp("[^xyz]", "g") or simply /\[^xyz]/g
  example > Do a global search for characters that are NOT "i" and "s" in a string:
  var str = "Do you know if this is all there is?";
  var patt1 = /[^is]/gi; 
  example > Do a global search for the character-span NOT from lowercase "a" to lowercase "h" in a string:
  var str = "Is this all there is?";
  var patt1 = /[^a-h]/g;
  example > Do a global search for the character-span NOT from uppercase "A" to uppercase "E":
  var str = "I SCREAM FOR ICE CREAM!";
  var patt1 = /[^A-E]/g; 
  example > Do a global search for the character-span NOT from uppercase "A" to lowercase "e":
  var str = "I Scream For Ice Cream, is that OK?!";
  var patt1 = /[^A-e]/g; 
  example > Do a global, case-insensitive search for the character-span that's NOT [a-s]:
  var str = "I Scream For Ice Cream, is that OK?!";
  var patt1 = /[^a-s]/gi; 
- The [^abc] expression is used to find any character NOT between the brackets.
- The characters inside the brackets can be any characters or span of characters:
  > [abcde..] - Any character between the brackets
  > [A-Z] - Any character from uppercase A to uppercase Z
  > [a-z] - Any character from lowercase a to lowercase z
  > [A-z ]- Any character from uppercase A to lowercase z
- Tip: Use the [abc] expression to find any character between the brackets.

# [0-9] - new RegExp("[0-9]") or simply /[0-9]/
  syntax with modifieres > new RegExp("[0-9]", "g") or simply /\[0-9]/g
  example > Do a global search for the number "1" in a string:
  var str = "12121212";
  var patt1 = /[1]/g; 
- The [0-9] expression is used to find any character between the brackets.
- The digits inside the brackets can be any numbers or span of numbers from 0 to 9.
- Tip: Use the [^0-9] expression to find any character that is NOT a digit.

# [^0-9] - new RegExp("[^0-9]") or simply /[^0-9]/ - support all 
  syntax with modifiers > new RegExp("[^0-9]", "g") or simply /\[^0-9]/g
  example > Do a global search for numbers that are NOT "1" in a string:
  var str = "12121212";
  var patt1 = /[^1]/g; 
  example > Do a global search for numbers that are NOT 5 to 8 in a string:
  var str = "123456789";
  var patt1 = /[^5-8]/g; 
- The [^0-9] expression is used to find any character that is NOT a digit.
- The digits inside the brackets can be any numbers or span of numbers from 0 to 9.
- Tip: Use the [0-9] expression to find any character between the brackets that is a digit.

# (x|y) - new RegExp("(x|y)") or simply /(x|y)/ - support all 
  syntax with modifier > new RegExp("(x|y)", "g") or simply /(x|y)/g
  example > Do a global search to find any of the specified alternatives (0|5|7):
  var str = "01234567890123456789";
  var patt1 = /(0|5|7)/g; 
- The (x|y) expression is used to find any of the alternatives specified.
- The alternatives can be of any characters.

# \b - new RegExp("\\bregexp") or simply /\bregexp/ - support all 
  syntax with modifiers > new RegExp("\\bregexp", "g") or simply /\bregexp/g
  example > Do a search for "LO" at the beginning of a word in a string:
  var str = "HELLO, LOOK AT YOU";

  example > Do a search for "LO" at the END of a word in a string:
  var str = "HELLO, LOOK AT YOU";
  var patt1 = /LO\b/;
- The \b metacharacter is used to find a match at the beginning or end of a word.
- Search for the pattern at the beginning of a word like this: \bLO
- Search for the pattern at the end of a word like this: LO\b
- If no match is found, it returns null.

# \B - new RegExp("\\Bregexp") or simply /\Bregexp/ - support all

  var str = "HELLO, LOOK AT YOU";

- The \B metacharacter is used to find a match, but where it is NOT at the beginning/end of a word.
- Search for the pattern NOT at the beginning of a word like this: \BLO
- Search for the pattern NOT at the end of a word like this: LO\B
- If no match is found, it returns null.

Quantifiers
--------------------------------------------------------------------------------
# + - new RegExp("n+") or simply /n+/ - support all 
  syntax with modifiers > new RegExp("n+", "g") or /\n+/g
  example > Do a global search for at least one "o":
  var str = "Hellooo World! Hello W3Schools!";
  var patt1 = /o+/g; 
  example > Do a global search for at least one word character:
  var str = "Hellooo World! Hello W3Schools!";
  var patt1 = /\w+/g; 
- The n+ quantifier matches any string that contains at least one n.

# * - new RegExp("n*") or simply /n*/ - support all 
  syntax with modifier > new RegExp("n*", "g") or simply /\n*/g
  example > Do a global search for an "l", followed by zero or more "o" characters:
var str = "Hellooo World! Hello W3Schools!";
var patt1 = /lo*/g; 
  example > Do a global search for a "1", followed by zero or more "0" characters:
var str = "1, 100 or 1000?";
var patt1 = /10*/g;
- The n* quantifier matches any string that contains zero or more occurrences of n.

# ? - new RegExp("n?") or simply /n?/ - support all 
  syntax with modifiers > new RegExp("n?", "g") or simply /\n?/g
  example > Do a global search for a "1", followed by zero or one "0" characters:
  var str = "1, 100 or 1000?";
  var patt1 = /10?/g;
- The n? quantifier matches any string that contains zero or one occurrences of n.

# {X} - new RegExp("n{X}") or simply /n{X}/
  syntax with modifiers > new RegExp("n{X}", "g") or simply /\n{X}/g
  example > Do a global search for a substring that contains a sequence of four digits:
  var str = "100, 1000 or 10000?";
  var patt1 = /\d{4}/g;
- The n{X} quantifier matches any string that contains a sequence of X n's.
- X must be a number.

# {X, y} - new RegExp("n{X, Y}") or /n{X, Y}/
  syntax with modifiers > new RegExp("n{X, Y}", "g") or /\n{X, Y}/g
  example > Do a global search for a substring that contains a sequence of three to four digits:
  var str = "100, 1000 or 10000?";
  var patt1 = /\d{3,4}/g;
- The n{X,Y} quantifier matches any string that contains a sequence of X to Y n's.
- X and Y must be a number.

# {X,} - new RegExp("n{X,}") or simply /n{X,}/
  syntax with modifier > new RegExp("n{X,}", "g") or simply /\n{X,}/g
  example > Do a global search for a substring that contains a sequence of at least three digits:
  var str = "100, 1000 or 10000?";
  var patt1 = /\d{3,}/g;
- The n{X,} quantifier matches any string that contains a sequence of at least X n's.
- X must be a number.

# $ - new RegExp("n$") or simply /n$/
  syntax with modifiers > new RegExp("n$", "g") or simply /\n$/g
  example > Do a global search for "is" at the end of a string:
  var str = "Is this his";
  var patt1 = /is$/g; 
  example > Do a global, multiline search for "is" at the end of each line in a string:
  var str = "Is\nthis\nhis\n?";
  var patt1 = /is$/gm; 
- The n$ quantifier matches any string with n at the end of it.
- Tip: Use the ^n quantifier to match any string with n at the BEGINNING of it.

# ^ - new RegExp("^n") or /^n/
  syntax with nodifiers > new RegExp("^n", "g") or simply /\^n/g
  example > Do a global search for "Is" at the beginning of a string:
  var str = "Is this his";
  var patt1 = /^Is/g; 
  example > Do a global, case-insensitive, multiline search for "is" at the beginning of each line in a string:
  var str = "\nIs th\nis h\nis?";
  var patt1 = /^is/gmi; 
- The ^n quantifier matches any string with n at the beginning of it.
- Tip: Use the n$ quantifier to match any string with n at the END of it.

# ?= new RegExp("regexp(?=n)") or simply /regexp(?=n)/
  syntax with modifiers > new RegExp("regexp(?=n)", "g") or simply /\regexp(?=n)/g
  example > Do a global search for "is" followed by " all":
  var str = "Is this all there is";
  var patt1 = /is(?= all)/g;
- The ?=n quantifier matches any string that is followed by a specific string n.
- Tip: Use the ?!n quantifier to match any string that is NOT followed by a specific string n.

# ?! - new RegExp("regexp(?!n)") or simply /regexp(?!n)/
  syntax with modifiers > new RegExp("regexp(?!n)", "g") or simply /\regexp(?!n)/g
  example > Do a global, case insensitive search for "is" not followed by " all":
  var str = "Is this all there is";
  var patt1 = /is(?! all)/gi; 
- The ?!n quantifier matches any string that is not followed by a specific string n.
- Tip: Use the ?=n quantifier to match any string that IS followed by a specific string n.

# constructor - RegExpObject.constructor - ECMAScript1
  return  > function RegExp() { [native code] }
  example > The constructor property returns a regular expression's constructor function:
  var patt = new RegExp("Hello World", "g");
  var res = patt.constructor; 
- In JavaScript, the constructor property returns the constructor function for an object.
- The return value is a reference to the function, not the name of the function:
- For JavaScript regular expressions the constructor property returns function RegExp() { [native code] }
- For JavaScript numbers the constructor property returns function Number() { [native code] }
- For JavaScript strings the constructor property returns function String() { [native code] }

# property - RegExpObject.global - ECMAScript1
  return  > (boolean) - returns true if the "g" modifier is set, false otherwise 
  example > Check whether or not the "g" modifier is set:
  var str = "Visit W3Schools!";
  var patt1 = /W3S/g;
  var res = patt1.global; 
- The global property specifies whether or not the "g" modifier is set.
- This property returns true if the "g" modifier is set, otherwise it returns false.

# ignoreCase - RegExpObject.ignoreCase - ECMAScript1
  return > (boolean) - returns (true) if the (i) modifier is set, (false) otherwise 
  example > Check whether or not the "i" modifier is set:
  var str = "Visit W3Schools!";
  var patt1 = /W3S/i;
  var res = patt1.ignoreCase; 
- The ignoreCase property specifies whether or not the "i" modifier is set.
- This property returns true if the "i" modifier is set, otherwise it returns false.

# lastIndex - RegExpObject.lastIndex - ECMAScript1
  return  > (number) - an integer that specifies the character position immediately after the last match found 
           by exec() or test() methods 
  example > o a global search for "ain" in a string, and output the index after a match is found:
  var str = "The rain in Spain stays mainly in the plain";
  var patt1 = /ain/g;
  while (patt1.test(str) == true) {
    document.write("'ain' found. Index now at: "+patt1.lastIndex);
    document.write("< br >");
  } 
- The lastIndex property specifies the index at which to start the next match.
- Note: This property only works if the "g" modifier is set.
- This property returns an integer that specifies the character position immediately after the last match found 
  by exec( ) or test( ) methods.
- Note: exec( ) and test( ) reset lastIndex to 0 if they do not get a match.

# multiline - RegExpObject.multiline - ECMAScript1
  return  > (boolean) - returns (true) if the (m) modifier is set, (false) otherwise 
  example > Check whether or not the "m" modifier is set:
  var str = "Visit W3Schools!";
  var patt1 = /W3S/gi; // "g" and "i" is set, "m" is not.
  var res = patt1.multiline; 
- The multiline property specifies whether or not the m modifier is set.
- This property returns true if the "m" modifier is set, otherwise it returns false.

# source - RegExpObject.source - ECMAScript1
  return  > (string) - the text of the RegExp pattern 
  example > Return the text of the RegExp pattern:
  var str = "Visit W3Schools";

# compile() - RegExpObject.compile(regexp, modifier) 
  support   > ([abc] expression) - all, but (opera) not support
  parameter > (regexp) - a regular expression 
  parameter > (modifier) - specifies the type of matching. (g) for a global match, (i) for a case-insensitive 
              match and (gi) for a global, case-insensitive match 
  example   > Do a global search for "man" in a string, and replace it with "person". Then change the regular 
              expression and replace either "man" or "woman" with "person", with the compile() method:
  var str = "Every man in the world! Every woman on earth!";
  var patt = /man/g;
  var str2 = str.replace(patt,"person");
  document.write(str2 + "< br >");
  patt = /(wo)?man/g;
  patt.compile(patt);
  str2 = str.replace(patt, "person");
  document.write(str2); 
- The compile() method was deprecated in JavaScript version 1.5.
- The compile() method is used to compile a regular expression during execution of a script.
- The compile() method can also be used to change and recompile a regular expression.

# exec() - RegExpObject.exec(string) - ECMAScript1
  parameter > (string) - required. the string to be searched 
  return    > (array) - an array containing the matched text if it finds a match, otherwise it returns (null) 
  example   > Search a string for the character "e":

  var res = patt.exec(str);

- The exec() method tests for a match in a string.
- This method returns the matched text if it finds a match, otherwise it returns null.
