# javascript variables can hold many data-types (numbers, strings, objects, and more) 
- like > var length = 16; var lastName = "Johnson"; var x = { firstName: "John", lastName: "Deo" };

# concept of data-types 
- in programming, data-types is an important concept 
- to be able to operatoe on variables, it is important to know something about the type 
  without data-types, a computer cannot safely solve, like > var x = 16 + "Volvo";
  does it make any sense to add (Volvo) to (16)? will it produce an error or will it produce a result?
# when adding a number and a string, javascript will treat the number as a string 
- javascript evaluates expressions from (left to right)
  so, different sequences can produce different results 	
- like > var x = 16 + 4 + "Volvo"; > 20Volvo
  like > var x = "Volvo" + 16 + 4; > Volvo164
	in the first example, javascript treats 16 and 4 as numbers, until it reaches (Volvo) 
	in the second example, since th efirst operand is a string, all operands are treated as strings 
	
# dynamic types 
- javascript has dynamic types, this means that the same variable can be used to hold different data-types 
- like > var x; x = 5; | x = "John";
	
# the (typeof) operator 
- you can use the javascript (typeof) operator to find the type of a javascript variable 
- the (typeof) operator returns the type of a variable or an expression

# strings - 
- a string (or a text string) is a series of characters
- strings are written with quotes, you can use (singel or double) quotes 
  like > var charNameOne = "Volvo"; | var charNameTwo = 'Volvo';
- you can use quotes inside a string, as long as they don't match the quotes surrounding the string 
  like > var strOne = "It's alright";
  like > var strTwo = "he is called 'Johnny'";
  like > var strThree = 'he is called "Johnny"';
	
# numbers - 
- javascript has only one type of numbers 
- numbers can be written with or without decimals 
  like > var x = 35.00 | var y = 35
- extra large or extra small numbers can be written with scientific (exponential) notation
  like > var x = 123e5 (12300000) | var y = 123e-5 (0.00123)
	
# booleans 
- booleans can only have two values (true or false) 
- booleans are often used in conditional testing

# javascript arrays 
- javascript arrays are written with square brackets 
- array items are separated by commas 
- the following code declares (creates) an array called (cars), containing three items

- javascript object are writtent with curly braces {}

  like > var person = { firstName: "John", lastName: "Doe", age: 50 };
- the object (person) in the example above has 3 properties (firstName, lastName, age)

# primitive types 
- a (primitive) data value is a single simple data value with no additional properties and methods 
- the (typeof) operator can return one of these primitive types (string, number, boolean, undefined) 
  like > typeof "John" > string 
  like > typeof 3.15 > number 
  like > var x; typeof x > undefined 
	
# complex types
- the (typeof) operator can return one of two complex types (function, object) 
- the (typeof) operator returns (object) for (object, arrays, and null) 
- the (typeof) operator does not return (object) for (functions) 
  like > typeof { name: "John", age: 35 } > object 
  like > typeof [1, 2, 3, 4] > object (not array is object) 
  like > typeof null > object 
  like > typeof function funcOne() { } > function
- the (typeof) operator returns (object) of (arrays) because in javascript (arrays) are (objects)

# undefined 
- in javascript, a variable without a value, has the value (undefined) the type is also (undefined) 
- any variable can be emptied, by setting the value to (undefined), the type will also be (undefined) 
  like > var char; (value is undefined and type is undefined) 
  like > char = undefined; (value is undefined and type is undefined) 
	
# null 
- in javascript (null) is nothing. it is supposed to be something that doesn't exist 
- unfortunately, in javascriptm, the data type of (null) is an object 
- note > you can consider it a bug in javascript that (typeof null) is an object, it should be (null) 

- you can empty an object by setting it to (null) 
  like > var person = { firstName: "John", lastName: "Deo" }; person = null; (now value is null, type is object)
- you can also empty an object by setting it to (undefined) 
  like > ver person = { firstName: "John", lastName: "Doe" }; person = undefined; (now value and type is undefined)	
	
# difference between (undefined and null) 
- (undefined) and (null) are equal in value but different in type 
  like > typeof undefined > undefined 
  like > typeof unll > object 
  like > null === undefined > false (type false)
  like > null == undefined > true (value true) 
	
# empty 
- an empty value has nothing to do with (undefined) 
- an empty string has both a legal value and a type 
  like > var car = ""; (value is "", and the typeof is string)

# javascript data-types 
- in javascript there are five (5) different data-types that can contain values, 
  these are (string, number, boolean, object, function) 
- there are six (6) types of object, 
  these are (object, date, array, string, number, boolean) 
- and two (2) data-types that cannot contain values,
  these are (null and undefined) 
	
# the (typeof) operator 
- you can use the (typeof) operator to find the data-type of a javascript variable 
  like > typeof "John"                  // returns string 
	like > typeof 3.5                     // returns number 
	like > typeof NaN                     // returns number 
	like > typeof false                   // returns boolean 
	like > typeof [1, 2, 3, 4, 5]         // returns object 
	like > typeof {name: "John", age: 35} // returns object 
	like > typeof new Date()              // returns object 
	like > typeof function() { }          // returns function 
	like > typeof myCar                   // returns undefined 
	like > typeof null                    // returns object 
	
- please observer - 
- the data-type of (NaN) is number 
- the data-type of an (array) is object 
- the data-type of a (date) is object 
- the data-type of (null) is object 
- the data-type of an (undefined) variable is (undefined*)
- the data-type of a (variable) that has not been assigned a value is also (undefined*)
- you cannot use (typeof) to determine if a javascript object is an array (or a data) 

# the data-type of (typeof) 
- the (typeof) operator is not a variable 
  it is an operator, operators (+ - * /) do not have any data-type 
- but, the (typeof) operator always (returns a string) containing the type of the operand 

# the (constructor) property 
- the (constructor) property returns the constructor function for all javascript variables 
  like > "John".constructor                  // returns function string() {[native code]}
  like > (3.15).constructor                  // returns function number() {[native code]}
  like > false.constructor                   // returns function boolean() {[native code]}
  like > [1, 2, 3, 4, 5].constructor         // returns function array() {[native code]}
  like > {name: 'John', age: 35}.constructor // returns function object() {[native code]}
  like > new Date().constructor              // returns function date() {[native code]}
  like > function() { }.constructor          // returns function function() {[native code]}
- you can check the (constructor) property to find out if an object is an (array) contains the word (array) 
  like > function isArray(myArray) { return myArray.constructor.toString().indexOf("Array") > -1; 
- or even simpler, you can check if th eobject is an (array function) 
  like > function isArray(myArray) { return myArray.constructor === Array; } 
- you can check the (constructor) property to find out if an object is a (date) contains the word (Date)
  like > function isDate(myDate) { return myDate.constructor.toString().indexOf("Date") > -1; }
- or even simpler, you can check if th eobject is a (Date function) 
  like > function isDate(myDate) { return myDate.constructor === Date; } 
	
# javascript type-conversion 
- javascript variables can be converted to a new variable and another data-type

# converting number to strings 
- the global method (string()) can convert numbers to strings 
- it can be used on any type of numbers, literals, variables, or expressions - 
  like > String(x) // returns a string from a number variable (x) 
	like > String(123) // returns a string from a number literal 123 
  like > String(100 + 23) // returns a string from a number from an expression 
- the number method (toString()) does the same 
  like > x.toString() | (123).toString() | (100 + 23).toString() 
- in the chapter (number method), you will find more methods that can be used to convert numbers to strings 
  1. toExponential() (returns a string, with a number rounded and written using exponential notation)
  2. toFixed() (returns a string, with a number rounded and written with a specified number of decimals)
  3. toPrecision() (returns a string, with a number written with a specified length)
	
# converting booleans to strings 
- the global method (string()) can convert booleans to strings 
  like > String(false) // returns false 
  like > String(true) // returns true 
- the boolean method (toString()) does the same 
  like > false.toString() // returns false 
  like > true.toString() // returns true 
	
# converting dates to strings 
- the global method (String()) can convert dates to strings 
  like > String(Date()) // returns "Thu Jul 17 2014 15:38:19 GMT+0200 (W. Europe Daylight Time)"
- the date method (toString()) does the same 
  like > Date().toString() // returns "Thu Jul 17 2014 15:38:19 GMT+0200 (W. Europe Daylight Time)"
- in the chapter (date methods), you will find more methods that can be used to convert dates to strings 
  1. getDate() (get the day as a number (1-31))
  2. getDay() (get the weekday a number (0-6))
  3. getFullYear() (get the four digit year (yyyy))
  4. getHours() (get the hour (0-23))
  5. getMilliseconds() (get the milliseconds (0-999))
  6. getMinutes() (get the minutes (0-59))
  7. getMonth() (get the month (0-11))
  8. getSeconds() (get the seconds (0-59))
  9. getTime() (get the time (milliseconds since January 1, 1970))
	
# converting strings to numbers 
- the global method (number()) can convert strings to numbers 
- strings containing numbers (like "3.14") convert to numbers (like 3.14) 
- empty strings convert to 0 
- anything else converts to (NaN) not a number 
  like > Number("3.14")  // returns 3.14
  like > Number(" ") // returns 0 
  like > Number("")  // returns 0 
  like > Number("99 88") // returns (NaN) 
- in the chapter (number methods), you will find more methods that can be used to convert strings to numbers 
  1. parseFloat() (parses a string and returns a floating point number) 
  2. parseInt() (parses a string and returns an integer) 
	
# the (unary +) operator 
- the (unary +) operator can be used to convert a variable to a number 
  like > var y = "5"  // y is a string 
  like > var x = + y; // x is a number 
- if the variable cannot be converted, it will still become a number, but with the value (NaN) not a number 
  like > var y = "John"; // y is a string 
  like > var x = + y; // x is a number (NaN) 
	
# converting booleans to numbers 
- the global nethod (Number()) can also convert booleans to numbers 
  like > Number(false) // returns 0 
  like > Number(true) // returns 1 
	
# converting Dates to numbers 
- the global method (Number()) can be used to convert dates to numbers 
  like > d = new Date(); Number(d) // returns 404568027739
- the date method (getTime()) does the same 
  like > d = new Date(); d.getTime() // returns 1404568027739
	
# automatic type conversion 
- when javascript tries to operate on a wrong data-type, it will try to convert the value to a right type 
- the result is not always what you expect 
  like > 5 + null // returns 5, because (null) is converted to 0
  like > "5" + null // returns "5null", because (null) is converted to "null" 
  like > "5" + 2  // returns "52", because 2 is converted to "2" 
  like > "5" - 2 // returns 3, because "5" is converted to 5 
  like > "5" * "2" // returns 10, because "5" and "2" are converted to 5 and 2 	
	
# automatic string conversion 
- automatically calls the variable's (toString()) function when  you try to output an object or a variable 
  like > document.getElementById("demo").innerHTML = myVar; 
  // if myVar = {name: "fjohn"} // toString converts to "[object Object]"
  // if myVar = [1,2,3,4]  // toString converts to "1,2,3,4"
  // if myVar = new Date()  // toString converts to "Fri Jul 18 2014 09:08:55 GMT+0200"
- numbers and booleans are also convertd, but this is not very visible 
  // if myVar = 123 // toString converts to "123"
  // if myVar = true // toString converts to "true"
  // if myVar = false // toString converts to "false"
